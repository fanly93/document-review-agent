## 三、关键流程设计发现

### 3.1 文件上传：最佳实践模式

**业界主流设计**：
- 格式支持：PDF / Word / Excel / 主流图片格式为标配，至少支持 10 种格式
- 大文件处理：**S3 预签名 URL 直传 + 断点续传**是云端产品标配（AWS 方案）
- 实时进度：**WebSocket 推送分阶段进度**（上传→OCR→提取→审核）优于轮询（合合信息方案）
- 错误恢复：单页失败不影响整批、自动多引擎降级处理

**差异化设计**：
- 合合信息：自动识别文件类型、纠正扩展名错误，用户体验最友好
- AWS Textract：页面级别错误隔离，处理精度最高

### 3.2 文档解析：技术栈选型建议

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 中文法律文档 OCR | PaddleOCR + 合合信息 | 中文精度高、法律术语适配 |
| 标准 PDF 文本提取 | PyMuPDF / pdfplumber | 速度快、精度稳定 |
| 复杂表格提取 | AWS Textract / Azure Forms | 表格识别能力最强 |
| 扫描件处理 | ABBYY / Tesseract | 高精度扫描文档处理 |
| 非结构化多格式 | Unstructured.io | 30+ 格式统一接口 |

**关键设计原则**：多引擎分层策略 → 先尝试直接文本提取，失败时升级 OCR，再失败则降级人工

### 3.3 自动审核（Agent）：三层架构模型

竞品调研揭示了最优的 **3 层自动审核架构**：

```
Layer 1: 快速检查（< 1s）
  → 格式校验、元数据提取、文档分类
  → 高置信度 → 直接通过
  
Layer 2: 结构化审核（3-10s）
  → 条款识别、规则引擎匹配、风险初筛
  → 置信度 ≥ 70% → 输出结果
  
Layer 3: 深度 LLM 分析（10-30s）
  → 复杂条款理解、跨条款关联、合规判断
  → 置信度 < 50% 或高风险 → 触发 HITL
```

**HITL 触发规则**（业界共识）：
- 风险等级 = Critical/High → 强制人工审核
- 置信度 < 50% → 转人工
- 未知文档类型 → 转人工
- 用户手动标记质疑 → 转人工

### 3.4 审核状态机：10 状态完整模型

```
uploaded → parsing → parse_failed (终态)
         ↓
      parsed → auto_reviewing → auto_review_failed
                              ↓
                         auto_reviewed → [直接完成 / 触发HITL]
                                       ↓
                              human_reviewing → human_review_failed
                                             ↓
                                          completed (终态)
                                          
任意状态 → rejected (终态，含审核回退原因)
```

关键指标：
- SLA 监控：每 5 分钟检查一次超期任务
- 超期催办：< 30 分钟触发通知，超期则自动重分配

## 四、关键结果呈现设计发现

### 4.1 风险呈现：分层金字塔是最优模式

Kira Systems 和 Luminance 共同验证了**4 层递进式信息架构**：

```
Level 1 → 整体风险得分（0-100，红橙黄绿色阶）
Level 2 → 分类风险看板（条款类型 × 风险权重）
Level 3 → 详细风险列表（可排序、可过滤）
Level 4 → 单条款深度页（置信度 + 原文 + 解释 + 建议）
```

**色彩规范**（业界标准）：
- Critical（8-10 分）：深红 `#DC143C`
- High（6-7 分）：橙红 `#FF4500`
- Medium（4-5 分）：橙色 `#FFA500`
- Low（2-3 分）：黄色 `#FFE000`
- Minimal（0-1 分）：绿色 `#90EE90`

### 4.2 原文定位：双视图同步是核心体验

**Adobe Acrobat AI + Kira Systems** 共同确立的最佳实践：
- 左侧：原文 PDF（带高亮标注）
- 右侧：审核结果面板
- 点击任意风险条目 → 左侧文档自动滚动到对应位置 + 高亮

定位精度要求：
- 段落级定位：基础要求
- 句子级定位：中级要求（大多数竞品支持）
- 字符级定位：高级要求（Adobe、Kira 支持）

### 4.3 来源追溯：透明度是用户信任的基础

**Perplexity 和 LexisNexis** 的来源展示模式值得借鉴：
- 行内引用标注（[1][2][3] 上标形式）
- 来源折叠展开面板（默认折叠，保持界面整洁）
- 法规/标准完整引用路径（如：《合同法》第 52 条第 3 款）
- 相似案例关联（法律专业场景特有）

### 4.4 解释性数据（XAI）：可解释 AI 是差异化关键

**Luminance Decipher** 的 XAI 实现最具参考价值：
- 置信度分级展示（事实提取 95% > 条款检查 70% > 风险评估 50%）
- 推理依据自然语言说明（"该条款被标记为高风险，因为：1. 缺少违约责任上限 2. 与第8条存在矛盾"）
- 决策树可视化（可选展开，面向法律专业用户）

### 5.3 技术选型建议

**文档处理栈**：
- 解析层：`unstructured.io` (多格式) + `PaddleOCR` (中文扫描件)
- 向量化：`text-embedding-3-large` 或等效模型
- 存储：对象存储（S3/OSS）+ PostgreSQL（元数据）+ 向量数据库（Qdrant/Weaviate）

**审核 Agent 栈**：
- 编排框架：`LangGraph`（支持状态机 + HITL + 可观测性）
- 模型：Claude claude-sonnet-4-6（深度分析）+ Claude claude-haiku-4-5（快速初筛）
- 可观测性：LangSmith

**前端呈现栈**：
- PDF 渲染：`PDF.js`（原文定位） + 自定义高亮层
- 双视图同步：虚拟滚动 + 位置锚点机制

## 六、竞品调研关键洞察

1. **流程可靠性 > 速度**：法律级别审核中，准确性和可追溯性优先于处理速度。竞品中 DocuWare 和 Kira Systems 的全审计日志是标配。

2. **人机协作不可省略**：即使是 AI 能力最强的 Luminance，HITL（人机交互循环）依然是核心架构组件，而非可选项。

3. **透明度是用户信任的根基**：用户最终是否采纳 AI 结论，取决于能否看到"为什么"——置信度、依据、来源缺一不可。

4. **定位精度决定使用效率**：原文定位不精准（只能定位到页面级）会让用户放弃使用结果面板，转而手动检索。字符级精准定位是高价值功能。

5. **分阶段产品路线**：
   - MVP：上传 + 解析 + 规则引擎审核 + 基础风险列表
   - V1：LLM 深度审核 + HITL + 双视图同步 + 来源引用
   - V2：XAI 解释性数据 + 热力图 + 团队协作 + 多角色权限
