## 二、各页面结构层次

### P01 — 文档上传页 `/upload`

```
页面根
├── 顶部导航栏
│   ├── 系统名称
│   └── 用户信息（角色/退出）
├── 页面主体
│   ├── 上传区域组件（UploadZone）
│   │   ├── 拖拽区（支持多文件，最多10个/次）
│   │   ├── 点击选择按钮
│   │   └── 支持格式说明文字（PDF / .docx / .doc）
│   ├── 文件预览列表（FilePreviewList）
│   │   ├── 文件名 + 大小 + 格式图标
│   │   ├── 单文件移除按钮
│   │   └── 上传进度条（分片上传时展示，0–40%）
│   └── 操作区
│       ├── 提交上传按钮（所有文件校验通过后可点击）
│       └── 清空列表按钮
└── 全局错误提示（Toast / 模态弹窗）
```

**前端校验规则（客户端拦截，不提交后端）**：
- 格式不支持：仅允许 `.pdf`、`.docx`、`.doc`，其余直接拦截并提示
- 文件大小超限：单文件 > 50MB 拦截
- 融资股权文件类型：由后端识别后返回硬拦截，前端展示专项提示

### P02 — 解析进度页 `/tasks/:taskId/parsing`

```
页面根
├── 顶部导航栏
├── 页面主体
│   ├── 文档信息卡片（文件名、上传时间）
│   ├── 进度可视化组件（ParseProgressPanel）
│   │   ├── 阶段图标列：上传中 → 文本提取 → 质量检测 → 完成
│   │   ├── 全局进度条（0–100%，WebSocket 驱动）
│   │   └── 当前阶段文案（动词进行时）
│   └── OCR 质量警告区（条件展示）
│       ├── 质量分显示（70–84% 时展示"解析质量中等"橙色警告）
│       └── 降级人工通道入口（质量分 < 70% 时展示）
└── 错误状态区（parse_failed 时展示）
    ├── 错误说明文案
    ├── 重新上传按钮
    └── 联系支持入口
```

**WebSocket 事件订阅**：`upload_progress`、`parse_progress`、`quality_check`、`parse_complete`、`parse_failed`

### P03 — 自动审核进度页 `/tasks/:taskId/reviewing`

```
页面根
├── 顶部导航栏
├── 页面主体
│   ├── 文档信息卡片
│   ├── 三层审核进度组件（AutoReviewProgressPanel）
│   │   ├── Layer 1 状态行：图标 + 文案 + 进度权重 5%
│   │   │   └── "格式校验 & 文档分类中…"
│   │   ├── Layer 2 状态行：图标 + 文案 + 进度权重 35%
│   │   │   └── "条款识别 & 规则匹配中…"
│   │   └── Layer 3 状态行：图标 + 文案 + 进度权重 60%
│   │       └── "LLM 深度分析中…"
│   └── 约束说明（"AI 辅助初审，仅供参考"）
└── 错误状态区（auto_review_failed 时展示）
    ├── 失败原因文案
    ├── 手动重试按钮（最多1次）
    └── 升级人工入口
```

**约束**：
- 此页面禁止展示任何审核结论（包括局部结果）
- 禁止展示"手动触发 HITL"入口
- 所有进度由 WebSocket 推送，禁止前端轮询

### P04 — 审核结果页 `/tasks/:taskId/result`

```
页面根
├── 顶部导航栏
├── 页面主体（四层递进信息架构）
│   ├── Level 1：整体风险评分区（RiskScorePanel）
│   │   ├── 风险仪表盘（0–100，5色阶）
│   │   ├── 风险等级文字标签（低/中/高/严重）
│   │   └── AI 辅助说明（"本报告为 AI 辅助初审结果，不构成法律建议"）
│   ├── Level 2：分类风险看板（RiskCategoryBoard）
│   │   ├── 各条款类型的风险数量卡片
│   │   └── 点击卡片跳转至对应 Level 3 过滤视图
│   ├── Level 3：风险列表（RiskItemList）
│   │   ├── 过滤栏（按风险等级 / 条款类型）
│   │   ├── 排序控制（风险等级降序/条款类型）
│   │   └── 风险条目卡片列表
│   │       ├── 风险等级色标
│   │       ├── 风险描述摘要
│   │       ├── 置信度徽章（颜色由后端 confidence_category 决定）
│   │       └── 展开按钮（进入 Level 4）
│   └── Level 4：单条款详情（RiskItemDetail，展开式抽屉或行内展开）
│       ├── 完整风险描述
│       ├── 置信度类别说明（事实提取/条款检查/风险评估）
│       ├── 原文定位（页码 + 段落，点击跳转至 HITL 双视图）
│       ├── AI 推理说明（低置信时必显）
│       └── 来源引用折叠面板
│           ├── 法规完整引用路径（如《合同法》第52条第3款）
│           └── 行内上标引用标注
├── 侧边栏：事实字段结构化区（FactExtractionPanel）
│   ├── 合同主体
│   ├── 合同金额
│   ├── 合同期限
│   └── 关键日期
└── 底部操作区
    ├── 导出报告按钮（MVP 可选）
    └── 查看审计日志入口
```

### P05 — 人工审核（HITL）页 `/tasks/:taskId/human-review`

```
页面根
├── 顶部工具栏（HumanReviewToolbar）
│   ├── 文档名称
│   ├── 整体风险评分（只读）
│   ├── 处理进度计数（已处理 N / 总 M 条）
│   ├── 完成审核按钮（仅 Critical/High 全部处理后可点击）
│   └── 整体任务驳回按钮（触发二次确认弹窗）
├── 双视图主体（左60% / 右40%）
│   ├── 左侧：原文 PDF 渲染区（PDFViewer）
│   │   ├── PDF.js 渲染层
│   │   ├── 高亮标注层（叠加，颜色与风险等级一致）
│   │   ├── 缩放控件
│   │   └── 翻页控件
│   └── 右侧：AI 审核结论面板（ReviewPanel）
│       ├── Level 1：整体风险评分（只读）
│       ├── Level 2：分类风险看板（只读）
│       ├── Level 3：风险列表（可排序/过滤，点击条目驱动左侧定位）
│       └── Level 4：条款详情 + 操作区
│           ├── 同意按钮（approve）
│           ├── 编辑按钮（edit，弹出编辑表单）
│           │   ├── 可编辑：风险等级、风险描述、AI 推理说明
│           │   └── 只读：风险类型、原文定位、置信度
│           ├── 单条驳回按钮（reject_item，必填理由 ≥ 10 字符）
│           └── 批注按钮（annotate，独立附加，不影响处理状态）
└── 弹窗层
    ├── 整体任务驳回确认弹窗（必填原因 ≥ 20 字符 + 二次确认）
    └── 编辑表单弹窗
```

**原文定位交互**：点击右侧任意风险条目 → 左侧 PDF 自动滚动至对应段落，并用与风险等级对应颜色高亮。

## 三、页面跳转关系

```
P01（上传）
   │  提交上传成功
   ▼
P02（解析进度）
   │  parse_complete 事件收到
   ▼
P03（自动审核进度）
   │  auto_review完成 + 无HITL触发
   ├──────────────────────────► P04（审核结果）
   │  auto_review完成 + HITL触发
   ▼
P05（人工审核）
   │  完成审核
   ▼
P04（审核结果）

P02 / P03 ──► P08（失败处理）   [parse_failed / auto_review_failed]
P05 ──────── P08（失败处理）   [human_review_failed]
P08 ──────── P01（重新上传）   [parse_failed 时]

P06（任务列表）──► P07（任务详情）──► P04 / P05（根据状态）
```

**路由守卫规则**：
- P05（HITL 页）仅当任务状态为 `human_reviewing` 时可访问，否则重定向至 P07
- P04（结果页）仅当任务状态为 `completed` 时展示完整内容，其他状态重定向至 P07
- 已终态任务（`rejected`、`parse_failed`）在 P07 只读展示，不提供继续操作入口

## 四、核心交互规范

### 4.1 WebSocket 实时进度交互

- **订阅时机**：进入 P02、P03 时自动建立 WebSocket 连接，以 `task_id` 为 channel 订阅
- **断线重连**：指数退避重连，最大 3 次，失败后切换为轮询降级（每 5 秒，仅查询状态）
- **事件格式**（前端只读，不可修改）：
  ```json
  { "event": "parse_progress", "task_id": "xxx", "stage": "ocr", "progress": 55, "message": "文档解析中…" }
  ```
- **禁止行为**：在 P02/P03 页面使用定时器轮询替代 WebSocket

### 4.2 置信度颜色渲染

| 后端字段值 | 前端颜色 | 展示规则 |
|-----------|---------|---------|
| `fact`（≥90%） | 绿色 | 直接展示数值 |
| `clause`（70-89%） | 黄色 | 直接展示数值 |
| `legal`（<70%） | 橙色 | 展示数值 + 必须渲染 AI 推理说明文本 |

**约束**：置信度颜色由后端 `confidence_category` 字段决定，前端禁止根据数值自行计算类别。

### 4.3 HITL 双视图联动交互

- 点击右侧任意风险条目 → 调用 PDF.js 的页面跳转 + 段落高亮 API
- 高亮颜色与风险等级一致（Critical=红，High=橙，Medium=黄，Low=蓝，Info=灰）
- 高亮标注为叠加层，不修改 PDF 原始内容

### 4.4 完成审核按钮的启用条件

前端维护一个本地计数器，跟踪：
- 当前任务所有 Critical 和 High 风险条目的已处理数量（approve / edit / reject_item 均计入）
- 仅当「已处理数 = Critical+High 总数」时，"完成审核"按钮才从 disabled 切换为 active
- 点击后向后端发送 POST 请求，**后端须再次校验**，前端不可绕过后端校验

### 4.5 整体任务驳回交互

1. 点击"驳回整体任务"
2. 弹出模态弹窗，必填原因输入框（最小长度 20 字符），输入不足时提交按钮禁用
3. 用户确认后发送 API 请求
4. 后端返回成功 → 前端跳转至 P07，展示 `rejected` 终态

### 4.6 错误提示分级

| 错误类型 | 展示组件 | 示例场景 |
|---------|---------|---------|
| 轻量提示（可自动消失） | Toast（右上角，3秒） | 单条操作成功/失败 |
| 阻断性错误（需用户确认） | 模态弹窗 | 整体驳回、文件格式错误 |
| 页面级错误（流程中断） | 专属错误区块（P08） | parse_failed、auto_review_failed |

## 五、重要前端页面功能元素汇总

| 组件名 | 所在页面 | 功能描述 | 关键约束 |
|-------|---------|---------|---------|
| `UploadZone` | P01 | 拖拽/点击上传，多文件支持 | 前端格式/大小校验，≥20MB 分片上传 |
| `FilePreviewList` | P01 | 展示待上传文件列表及进度 | 分片进度由 WebSocket 驱动 |
| `ParseProgressPanel` | P02 | 解析阶段三步进度可视化 | 仅 WebSocket 驱动，禁止轮询 |
| `AutoReviewProgressPanel` | P03 | 三层审核进度（Layer1/2/3）分层展示 | 禁止展示部分结论，禁止手动触发 HITL |
| `RiskScorePanel` | P04 | 整体风险评分仪表盘（0-100, 5色阶） | 数据来自后端，前端只展示 |
| `RiskCategoryBoard` | P04 | 条款类型×风险数量看板 | 点击跳转过滤列表 |
| `RiskItemList` | P04/P05 | 可过滤/排序的风险条目列表 | 置信度颜色由后端字段决定 |
| `RiskItemDetail` | P04/P05 | 条款详情展开：推理说明+来源引用 | 低置信时必渲染推理说明 |
| `PDFViewer` | P05 | PDF.js 渲染 + 高亮叠加层 | 段落级精度（MVP），高亮颜色由风险等级决定 |
| `ReviewPanel` | P05 | 右侧 AI 结论面板（4层信息架构） | 与 PDFViewer 联动 |
| `HumanReviewToolbar` | P05 | 顶部工具栏，含完成/驳回按钮 | 完成按钮有启用条件守卫 |
| `OperationButtonGroup` | P05 | 同意/编辑/单条驳回/批注四类操作 | 每类操作均异步提交后端，按 reviewer_status 切换显示 |
| `TaskTable` | P06 | 任务列表表格，含状态色标 | 支持过滤/搜索/分页 |
| `StatusTimeline` | P07 | 任务状态流转时间线 | 只读展示，来自后端 |
| `AuditLogPanel` | P07 | 审计日志折叠区 | 只读，分页加载 |
| `FactExtractionPanel` | P04 | 结构化事实字段展示 | 置信度颜色标注（来自后端） |
